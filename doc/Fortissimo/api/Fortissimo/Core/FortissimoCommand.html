A Fortissimo command.

The main work unit in Fortissimo is the FortissimoCommand. A FortissimoCommand is expected to conduct a single unit of work -- retrieving a datum, running a calculation, doing a database lookup, etc. Data from a command (if any) can then be stored in the FortissimoExecutionContext that is passed along the chain of commands.

Each command has a request-unique name (only one command in each request can have a given name), a set of zero or more params, passed as an array, and a FortissimoExecutionContext object. This last object contains the results (if any) of previously executed commands, and is the depository for any data that the present command needs to pass along.

Typically, the last command in a request will format the data found in the context and send it to the client for display.

		execute (line 667)
	
 
	
	

Execute the command.


Typically, when a command is executed, it does the following: 
  • uses the parameters passed as an array.
  • performs one or more operations
  • stores zero or more pieces of data in the context, typically keyed by this
    object's $name.
  Commands do not return values. Any data they produce can be placed into  the FortissimoExcecutionContext object. On the occasion of an error,  the command can either throw a FortissimoException (or any subclass  thereof), in which case the application will attempt to handle the error. Or it  may throw a FortissimoInterrupt, which will interrupt the flow of the  application, causing the application to forgo running the remaining commands.

	
    
				
  • throws: FortissimoException   Thrown if the command experiences a general execution error. This may not   result in the termination of the request. Other commands may be processed after   this.
    • 
				
  • throws: FortissimoInterrupt   Thrown when the command should be treated as the last command. The entire   request will be terminated if this is thrown.
